# <a name="module-2-introduction-to-microsoft-dataverse"></a>課程模組 2：Microsoft Dataverse 簡介

# <a name="scenario"></a>案例

Bellows College 是一個教育組織，校園內有多棟大樓。 校園訪客造訪情況目前記錄在紙本日誌中。 此資訊並未以一致的方式擷取，而且也無法收集和分析整個校園造訪情形的相關資料。

校園行政單位想要更新其訪客登記系統，讓保全人員控管各棟大樓的出入狀況，且所有造訪情形都必須由大樓負責人預先登記和記錄。

在整個課程中，您將建立應用程式並執行自動化功能，以便 Bellows College 的行政和保全人員能夠管理及控管校內大樓的出入狀況。

在此實驗室中，您將存取自己的環境、建立 Microsoft Dataverse 資料庫，以及建立可追蹤變更的解決方案。 此外，您也會建立資料模型以支援下列需求：

-   R1：追蹤校園造訪情形的位置 (大樓)

-   R2：記錄基本資訊以識別並追蹤訪客

-   R3：排定、記錄及管理造訪情形

最後，您會將範例資料匯入至 Microsoft Dataverse。

# <a name="high-level-lab-steps"></a>高階實驗室步驟

為準備您的學習環境，您必須：

* 建立解決方案和發佈者
* 新增所需的新元件和現有元件，以符合應用程式需求。 如需中繼資料描述 (資料表和關聯性)，請參閱[資料模型文件](https://github.com/MicrosoftLearning/PL-900-Microsoft-Power-Platform-Fundamentals/blob/master/Allfiles/Campus%20Management.png)。 您可以按住 CTRL 並在連結上按一下滑鼠左鍵，或在連結上按一下滑鼠右鍵，以在新視窗中開啟資料模型文件。
* 建立大樓及造訪情形資料表
* 使用 Excel 試算表匯入造訪情形資料

# <a name="exercise-1-create-new-table"></a>練習 \#1：建立新資料表

**目標：** 在此練習中，您將會建立新的自訂資料表 Visits。 

## <a name="task--1-create-visit-table-and-columns"></a>工作 \# 1：建立「造訪情形」資料表和資料行

此 **造訪情形** 資料表將包含校園造訪情形的相關資訊，包括大樓、訪客、每個造訪情形的預定時間和實際時間。

我們想要在造訪簽入程序期間、當系統要求時，為每個造訪情形指派可讓訪客輕鬆輸入和理解的唯一號碼。

>   我們使用 **不受時區影響** 的行為來記錄日期和時間資訊，因為造訪時間一律使用大樓所在位置的當地時間，且從其他時區檢視時也不應變更。

1.  登入 <https://make.powerapps.com> (若您還未登入)

2.  如果尚未選取，請在右上方選取您的 [[我的縮寫名] 練習] 環境。

3.  使用左側導覽，展開 [Dataverse]，然後選取 [資料表]。

4.  按一下 [新增資料表]。

5.  輸入 **造訪情形** 作為 [顯示名稱]。

6.  按一下 [建立]。 此動作會開始在背景佈建資料表，同時您也可以開始新增其他資料行。

7.  建立「預定開始時間」資料行

    1.  您應該會在 [造訪情形資料表資料行] 頁面中。

    2.  請務必選取 **[資料行]** 索引標籤，然後按一下 **[新增資料行]** 。

    3.  輸入 **預定開始時間** 作為 [顯示名稱]。

    4.  選取 [日期和時間] 作為 [資料類型]。

    5.  在 **[必要項目]** 中，選取 **[必要]** 。

    6.  展開 **[進階選項]** 區段。

    7.  在 **[行為]** 中，選取 **[不受時區影響]** 。

    8.  按一下 [完成]。

8.  建立「預定結束時間」資料行

    1.  按一下 **[新增資料行]** 。

    2.  輸入 **排定的結束時間** 作為 [顯示名稱]。

    3.  選取 [日期和時間] 作為 [資料類型]。

    4.  在 **[必要項目]** 中，選取 **[必要]** 。

    5.  展開 **[進階選項]** 區段。

    6.  在 **[行為]** 中，選取 **[不受時區影響]** 。

    7.  按一下 [完成]。

9.  建立「實際開始時間」資料行

    1.  按一下 **[新增資料行]** 。

    2.  輸入 **實際開始時間** 作為 [顯示名稱]。

    3.  選取 [日期和時間] 作為 [資料類型]。

    4.  在 **[必要項目]** 中，將此項保留為 **[選用]** 。

    5.  展開 **[進階選項]** 區段。

    6.  在 **[行為]** 中，選取 **[不受時區影響]** 。

    7.  按一下 [完成]。

10. 建立「實際結束時間」資料行

    1.  按一下 **[新增資料行]** 。

    2.  輸入 **實際結束時間** 作為 [顯示名稱]。

    3.  選取 [日期和時間] 作為 [資料類型]。

    4.  在 **[必要項目]** 中，將此項保留為 **[選用]** 。

    5.  展開 **[進階選項]** 區段。

    6.  在 **[行為]** 中，選取 **[不受時區影響]** 。

    7.  按一下 [完成]。

11. 建立「代碼」資料行

    1.  按一下 **[新增資料行]** 。

    2.  輸入 **代碼** 作為 [顯示名稱]。

    3.  選取 [自動編號] 作為 [資料類型]。
    
    4.  選取 **[日期前置編號]** 做為 **[自動編碼類型]** 。

    5.  按一下 [完成]。

12. 建立 [訪客] 查詢資料行

    1.  按一下 **[新增資料行]** 。

    2.  輸入 **訪客** 作為 [顯示名稱]。

    3.  選取 [查詢] 作為 [資料類型]。

    4.  選取 [連絡人] 作為 [相關資料表]。

    5.  按一下 [完成]。

13. 按一下 **[儲存資料表]**

# <a name="exercise-2-import-data"></a>練習 \#2：匯入資料

**目標：** 在此練習中，您會將範例資料匯入至 Dataverse 資料庫。

## <a name="task-1-import-the-visitsxls-file"></a>工作 \#1：匯入 Visits.xls 檔案。

在此工作中，您將匯入包含完成資料匯入所需之 Power Automate 流程的解決方案。

1.  您應將 **Visits.xls** 檔案儲存在桌面上。 請下載 [Visits.xls](https://github.com/MicrosoftLearning/PL-900-Microsoft-Power-Platform-Fundamentals/raw/majorupdate_april2022/Allfiles/Visits.xlsx) 檔案 (若您尚未下載)。

2.  若尚未登入，請登入至 <https://make.powerapps.com>。

3.  如果尚未選取，請在右上方選取您的 [[我的縮寫名] 練習] 環境。

4.  使用左側導覽，展開 [Dataverse]，然後選取資料表。

5.  找到並開啟您在上一個練習中建立的 **Visit** 資料表。

6.  使用頂端的功能表，選取 [資料] 旁的下拉式箭號、選取 [取得資料] 旁的箭號，然後選取 [從 Excel 取得資料]。

7.  在隨後出現的功能表中，選取 [取得資料]，然後選取 [從 Excel 取得資料]。

8.  在隨後出現的功能表中，選取 [上傳] 按鈕。

9.  找到並選取您稍早已下載的 **Visits.xls** 檔案。 *(請注意：上傳檔案可能會需要一或二分鐘的時間。如果出現對應錯誤的訊息，請無需擔心，我們稍後會修正這些問題。)*

10. 選取 [對應資料行]。

11. 請依以下說明來對應資料行：

    | 瀏覽 Db 資料行 | 來源值   |
    |------------------|-----------------|
    | 實際結束       | 實際結束      |
    | 實際開始     | 實際開始    |
    | 程式碼             | 程式碼            |
    | 名稱             | 名稱            |
    | 計劃結束    | 排程結束   |
    | 排定的開始時間  | 排程開始 |

12. 將其餘所有欄位保留為 [未設定]。

13. 在畫面右上角，選取 [儲存變更]。

14. 在 [匯入資料] 畫面中，確認對應狀態顯示為 [對應成功]，然後選取 [匯入] 按鈕。

**注意：** 「將資料匯入資料表中，可能需要花費數鐘的時間。如果出現一些錯誤，請無需擔心，這是正常的情況且不會影響其他課程。」

## <a name="task-2-verify-data-import"></a>工作 \#2：確認資料匯入

1.  匯入資料後，請使用畫面左側的瀏覽，再次選取 [瀏覽] 資料表。

2.  使用位於頂端的索引標籤，選取 [資料] 索引標籤。

3.  確認資料表中有記錄。

恭喜，您已順利建立了新資料表並匯入資料。
